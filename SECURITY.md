# Security Policy

## Reporting a vulnerability

Suspected security issues should not be reported through a public issue.

Please use GitHub's private vulnerability reporting for this repository when it
is available. If that channel is unavailable, contact the maintainer directly
through GitHub and request a private coordination path.

The initial response target is seven days. The exact remediation timeline
depends on severity, exploitability, and the scope of the required fix.

## Scope

This policy covers the tracked code in this repository. It does not extend to:

- local benchmark artifacts that are intentionally omitted from version control
- third-party datasets and external benchmark archives
- upstream model providers and external forecasting packages

## Disclosure

Public disclosure should wait until the issue has been triaged and a fix,
mitigation, or explicit no-fix determination has been prepared.
